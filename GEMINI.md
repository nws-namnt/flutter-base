## Reference

This project is a Flutter production boilerplate based on [itachi1611/flutter_temp_bloc](https://github.com/itachi1611/flutter_temp_bloc). When adding new modules, follow the patterns established there.

## Commands

```bash
# Install dependencies
flutter pub get

# Run app (choose a flavor: dev | uat | prod)
flutter run --flavor dev

# Analyze
flutter analyze

# Run tests
flutter test

# Run a single test file
flutter test test/widget_test.dart

# Code generation (Retrofit, JSON serializable, Hive adapters)
flutter pub run build_runner build --delete-conflicting-outputs

# Watch mode for code generation
flutter pub run build_runner watch --delete-conflicting-outputs

# Build APK
flutter build apk --flavor dev --release

# Generate localization files
flutter pub run intl_utils:generate

# Firebase CLI ‚Äî login (required before flutterfire configure)
npm install -g firebase-tools   # install once
firebase login

# FlutterFire CLI (use fvm dart when using FVM)
fvm dart pub global activate flutterfire_cli
fvm dart pub global run flutterfire_cli:flutterfire configure \
  --project=<firebase-project-id> \
  --out=lib/firebase/<flavor>/firebase_options.dart \
  --platforms=android,ios

# Change package name (uncomment change_app_package_name in pubspec first)
flutter pub run change_app_package_name:main com.new.package.name
```

## Key conventions

**State management** ‚Äî use Cubit (not full Bloc) for all state. Each page owns its own Cubit, instantiated inside `initState` and closed in `dispose`. Global cubits (`AppCubit`, `HomeShellCubit`) are provided at `AppPage` level via `MultiBlocProvider`.

**Navigation** ‚Äî GoRouter with `StatefulShellRoute.indexedStack`. Always navigate using `context.go(Routers.x.routerPath)` or `context.goNamed(Routers.x.routerName)`. Never use `Navigator.push` directly. Add new routes to `Routers` enum first, then wire in `router_config.dart`.

**Flavors** ‚Äî the app supports three flavors: `dev`, `uat`, `prod`. Firebase options are selected at runtime via `appFlavor` constant. Always pass `--flavor <env>` when running or building.

**Theming** ‚Äî colors come from the TailwindCSS v3 palette defined in `app_themes.dart` (`primarySwatch` = indigo, `textSwatch` = slate). Never hardcode colors in widgets; reference `Theme.of(context).colorScheme` or the swatch constants.

**Code generation** ‚Äî Retrofit API clients, JSON models, and Hive adapters all require `build_runner`. Run after any change to files annotated with `@RestApi`, `@JsonSerializable`, or `@HiveType`.

**Localization** ‚Äî ARB files live in `l10n/`. After editing them run `intl_utils:generate` to regenerate `lib/generated/`. Use `S.of(context).keyName` in widgets.

**Logger** ‚Äî use `simpleLog()` for quick debug prints, `info()` / `warn()` / `err()` for structured logs. `AppFilter` can be toggled to allow only error/warning logs in production builds.

  ‚Üí FirebaseMessaging.onBackgroundMessage(handler)
  ‚Üí GetStorage.init()
  ‚Üí AppSharedPreference.init()

  ‚Üí build: MultiBlocProvider[AppCubit, HomeShellCubit]
         ‚Üí BlocBuilder<AppCubit> ‚Üí MaterialApp.router(theme, locale, routerConfig)

Router
  / (root)  ‚Üí SplashPage  ‚Üí context.go('/home')
  /home     ‚Üí HomeShellPage (StatefulShellRoute)
              ‚îú‚îÄ‚îÄ branch 0: HomePage
